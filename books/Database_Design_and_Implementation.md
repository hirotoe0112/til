## Chapter 1 Database Systems

- データベースのrequirements
  - persistent
  - can be shared
  - kept accurate
  - very large
  - usable
- データベースを永続化するにはテキストファイルに記録するのが一般的
  - 最もシンプルなやり方は1行ずつタブ区切りで書くこと
    ```
    1[tab]Tom[tab]25[ret]2[tab]Kate[tab]21
    ```
  - しかしこのやり方は更新効率が悪い、検索に時間がかかるという理由で非効率である
- データベースは不要な待ち時間を発生させないため複数ユーザが同時に使える必要がある
  - しかし同時にレコードを更新して片方しか反映されないということはあってはならない
  - コンフリクトが発生する操作を検知し片方を待たせる仕組みが必要
- データベースは以下の困難に直面する
  - RAM以上のデータを
  - ディスクという遅いデータを使って扱い
  - 複数ユーザが同時にアクセスする状況で
  - 完全に回復可能である必要があり
  - 合理的なレスポンスタイムを守る必要がある
- これらを実現するためにRAMへのキャッシュが使えるが、永続バージョンとRAMバージョンの一貫性を保つ必要がある
- JavaベースのDerbyデータベースの紹介
  - [Apache Derby](https://db.apache.org/derby/)
- UIとDBエンジンの接続方法には埋め込み型・サーバー型がある
  - 埋め込み型はUIとDBエンジンが一体となっておりそのUIからのみ利用されることを想定
  - サーバー型はUIとDBエンジンが分かれており複数のUIから利用される
- connection stringは埋め込み型か、サーバー型か、どのサーバーに接続しにいくかを完全にカプセル化している
  - connection stringを変更すれば、他のコマンド（selectなど）は影響を受けない
