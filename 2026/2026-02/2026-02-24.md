## GitHub Actions

- 定時実行するにはon scheduleでcron式を指定する

  ```yaml
  on:
    schedule:
      - cron: "0 15 * * *" # JST 0時に実行
  ```

  - ただし時間通り発火しない場合もあるらしい
  - 今回は単にその日のTILファイルを自動で作成してほしいだけなので、遅れても問題なし

- GitHub Actionsの中でコミットする際のメールアドレスはユーザ専用の`noreply`アドレスを使うのが一般的らしい
  - [コミットメールアドレスを設定する - GitHub ドキュメント](https://docs.github.com/ja/account-and-profile/how-tos/email-preferences/setting-your-commit-email-address#about-commit-email-addresses)
  - [Email addresses reference - GitHub Docs](https://docs.github.com/en/account-and-profile/reference/email-addresses-reference#your-noreply-email-address)
  - プロフィールのEmailsの画面に以下のように記載があった
    ```
    We’ll remove your public profile email and use xxxxxxxx@users.noreply.github.com when performing web-based Git operations (e.g. edits and merges) and sending email on your behalf.
    ```

  - Fear Of Missing Out
  - 取り残される恐怖を利用した手法
  - ↓みたいなやつもその一種？
