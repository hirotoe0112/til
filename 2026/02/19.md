## VS Code保存時整形

- 保存時整形の設定をするには`settings.json`に以下のような設定が必要

  ```json
  {
    "editor.formatOnSave": true,
    "editor.defaultFormatter": "esbenp.prettier-vscode",
    "[markdown]": {
      "editor.formatOnSave": true,
      "editor.defaultFormatter": "esbenp.prettier-vscode"
    },
    "prettier.requireConfig": false
  }
  ```

- markdownlintというのもある
  - [VSCodeでPrettierとMarkdownlintを併用し、マークダウンが変な整形するのを防ぐ #prettier - Qiita](https://qiita.com/satoshi256kbyte/items/5919c89df8a0587b486b)
- しかしテーブルをいい感じに整形してくれなかったり、なんだかんだPrettierが一番使い勝手が良さそう
  - [Markdownをフォーマッターで綺麗にしたい | 株式会社アイオス](https://www.ios-net.co.jp/blog/20240626-2847/)

## レイヤードアーキテクチャ

- ハンドラー層はHTTPリクエスト／レスポンスに関することに責任を持つ
  - HTTPリクエストが正しいかどうかのチェックとHTTPレスポンス返却
- サービス層はビジネスロジックに責任を持つ
- インフラ層はデータの永続化に責任を持つ
- これらの責任分離をしっかりすることで保守性の高いアーキテクチャを実現できる

## CloudWatchメトリクス

- API Gatewayのリクエスト数で、期間を1分にして`m1/PERIOD(m1)`の数式を入力すると1秒あたりのリクエスト数がグラフで表示できる
