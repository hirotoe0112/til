## NASM

- プログラムの書き方
  - Linux System Call Tableの`x86 (32-bit)`のテーブルを見る
    - [Linux System Call Table](https://www.chromium.org/chromium-os/developer-library/reference/linux-constants/syscalls/#x86-32-bit)
  - 例えばwriteをしたければ、4番のwriteを見る
  - 引数が`ebx`から`edx`まであるので、それぞれに値を`mov`で入れる
    ```
    mov edx, 13 ;number of byte
    mov ecx, msg ;constant to be written
    mov ebx, 1 ;write to stdout
    ```
  - `eax`でオペコードを指定する
    ```
    mov eax, 4
    ```
  - `int 80h`でシステムコールをして実行する。これは固定の文言で決まっているっぽい
    ```
    int 80h
    ```
  - プログラム終了時には`exit`を呼び出して終了する
    ```
    mov ebx, 0 ;exit code
    mov eax, 1
    int 80h
    ```
  - Hello World!を標準出力に出力するプログラムの全体は以下のような感じになる

    ```
    SECTION .data
    msg db 'Hello World!', 0Ah ;assign msg variable with your message string

    SECTION .text
    global _start ;tell the kernel where to begin execution

    _start:
      mov edx, 13 ;number of bytes to write
      mov ecx, msg ;move the memory address of our message string into ecx
      mov ebx, 1 ;write to stdout
      mov eax, 4 ;opcode
      int 80h ;system call

      mov ebx, 0 ;exit code
      mov eax, 1 ;call exit
      int 80h
    ```

- `ebx`とかはレジスタを指す。movで値をそのレジスタに入れている。操作を実行するとき、オペコードごとに、「この値を`ebx`レジスタから取ってきますよ」みたいに決まっている
- 文字列の長さを固定ではなく動的に指定するには、pointer arithmeticを使用する
  - 文字列のアドレスをeax, ebxの両方に入れておく。eaxが指す値と終端文字列（0）を比較する。終端ではない場合、eaxをインクリメントする。終端の場合、eaxからebxを引いたものが長さとなる
  - 複数の文字列を格納する場合、隣接したメモリに格納される。最初の文字列は、次の文字列の最後まで終端にならないことになる。これを防ぐために、終端文字列を挿入する必要がある
- サブルーチンを呼び出すのは`call`、戻るのは`ret`
