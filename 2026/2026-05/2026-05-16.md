## Claude Code

- 突然、`claude`コマンドを実行しても何も反応がなくなった
  - ⇒再インストールした
- サブエージェントを使ってブログを書かせる実験をしているリポジトリで、他の端末では問題なく動いていたが、家のPCではサブエージェントのwrite権限がないと言われた
  - バックグラウンドモード (run_in_background: true)で起動していたことが原因らしい（Claude Code談）
  - フォアグラウンドで起動するようエージェントメモリに追記した（Claude Codeが）

## Computer Science

- forward secrecy
  - 鍵が万が一漏洩した場合でも、過去のデータを守ろうとする考え方
  - [Forward Secrecy 入門](https://rms.ne.jp/sslserver/basis/forwardsecrecy/)

- かつてルーターはデータを保存するストレージがなかったため起動時にソフトや設定をどこかからとってくる必要があった
  - BOOTP(the bootstrap protocol)がソフトウェアローディングに使用された
  - しかし設定の方はBOOTPでは管理が難しかった
  - DHCPの登場
- DHCPリレーエージェント
  - DHCPサーバとクライアントが別にネットワークにあってもパケットが転送される仕組み
  - ネットワークごとにDHCPサーバを持たなくてよくなる
  - [DHCPリレーエージェントとは](https://www.infraexpert.com/study/tcpip14.html)
- BOOTPのプロセス
  - クライアントはブロードキャストしてBOOTPサーバのポート67が受信する
  - BOOTPサーバはクライアントIPアドレスとダウンロードファイルの場所（オプション）を返す
  - クライアントはダウンロードファイルの場所にTFTP（Trivial File Transfer Protocol）で取りに行く
  - TFTPサーバはポート69で受信してファイルを返す
